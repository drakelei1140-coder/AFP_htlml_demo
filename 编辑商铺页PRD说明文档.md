# 编辑商铺页 PRD 说明文档（详细版）

## 1. 文档信息
- 适用页面：`shop-edit.html`
- 页面定位：商铺编辑页（草稿 / 提交审核）
- 更新时间：2026-04-27

## 2. 页面目标
1. 支持对已存在商铺进行资料维护，反显已有信息并允许更新。
2. 保持编辑场景关键约束：不可修改关联企业、存量设备不可直接编辑。
3. 支持在编辑阶段追加新设备并填写数量与支付金额，自动汇总终端数量与总金额。

## 3. 页面结构
1. 顶部商铺信息卡（SID、商铺名称、营业状态、CID）
2. 模块锚点导航（12个模块）
3. 表单主体（企业关联、基础信息、名称、联系、地址、文件、设备、人员等）
4. 底部操作栏（保存草稿 / 提交审核 / 取消）
5. 弹窗区（选择服务、选择设备、选择已有人员）

## 4. 全局交互规则
- 编辑页打开后默认反显商铺现有数据。
- 锚点导航点击平滑滚动，当前模块按钮高亮。
- 关联企业信息为只读展示，不可修改。
- 提交审核执行完整校验；保存草稿执行轻量校验。
- 终端设备总量与总金额由“存量设备 + 新增设备”自动汇总。

## 5. 模块与字段清单

> 字段格式：**字段名｜控件类型｜说明**

### 5.1 顶部商铺信息卡
- SID｜只读展示｜反显已有SID
- 商铺名称｜只读展示｜反显已有名称
- 商铺营业状态｜只读展示
- CID｜只读展示

### 5.2 关联企业信息（只读）
- CID｜只读展示
- 企业名称｜只读展示
- 企业营业状态｜只读展示
- 企业类型｜只读展示
- 所属地区｜只读展示
- 关联群组｜只读展示
- 说明文案｜提示｜编辑商铺不可修改关联企业

### 5.3 主键 / 关联
- SID｜只读展示
- CID｜只读输入｜随关联企业反显

### 5.4 商铺基础
- 开通Kpay服务｜弹窗多选 + Tag｜反显后可调整
- 店面相片 / 存倉相片｜上传
- 租金 / 租約年期｜输入
- 私隱政策 / 產品目錄 / 送貨政策｜上传
- 门店类型 / 聯絡語言 / 商戶營業狀態(必須)｜下拉
- 特殊業務屬性標籤｜多选
- 退貨政策 / 退貨政策截圖｜输入+上传
- 審核類型 / 上單來源 / 服務類型｜输入/下拉
- 合作夥伴推薦人 / 合作夥伴編號｜输入
- 交單備註｜文本域
- 高額商戶 / 首次申請卡機 / 卡機品牌 / KPay占比 / 重複簽約原因｜输入/下拉

### 5.5 商铺名称
- 当地语言名称 / 简称｜输入（反显）
- 英文名称 / 简称｜输入（反显）
- 第三语言名称 / 简称｜输入
- 其它名称｜输入（反显）
- 所有名称（当地语言 / 英文）｜输入（反显）
- 切換商铺顯示名稱（中文 / 英文）｜下拉｜候选项由名称字段动态生成

### 5.6 商铺联系
- 固定電話 / 手提電話｜输入（反显）
- 商铺電郵地址 / 管理員電郵 / 運營電郵｜邮箱输入（反显）
- 商铺接收營銷資訊 / 接收營銷資訊｜下拉
- 商铺微信ID｜输入

### 5.7 商铺地址
- 主要營業地址（当地语言 / 英文）｜文本域（反显）
- 營業鋪位｜输入
- 營業國家 / 營業地區｜下拉
- 门店地址（城市 / 州省 / 地址行1-3）｜输入/下拉
- 郵寄地址 / 郵政編碼｜输入

### 5.8 线上资料
- Website(Desktop/Mobile)｜URL输入
- APP(Android/iOS)｜URL输入
- Facebook / Instagram / Openrice｜URL输入
- H5經營網址｜URL输入

### 5.9 商铺文件
- 租約證明 / 商铺LOGO / 營業環境照片三張 等｜文件上传
- 營業環境照片三張｜多文件上传｜最多 3 张
- 商業登記證號碼 / 名稱 / 有效期｜输入/日期
- 证照副本、证明附件等｜文件上传

### 5.10 门店经营特征
- 每宗交易平均金額 / 最大交易額 / 年交易宗數｜数值
- 過往拒付比例 / 退款比例｜百分比
- 拒付/退款平均處理日數｜数值
- 平均每月營業額 / 經營方式｜输入
- 提前收款日數 / 最短送達時間 / 最長送達時間｜输入
- 行業分類 / 業務性質 / 產品服務｜下拉
- 小費功能｜下拉

### 5.11 终端设备（编辑场景）
#### 5.11.1 列表结构
- 字段：设备型号｜设备分类｜设备序列号｜状态｜数量｜支付金额｜数据来源｜操作
- 数据来源｜标签文本｜区分“存量设备（只读）”与“本次新增”

#### 5.11.2 存量设备规则
- 存量设备行仅展示，不可修改数量/金额，不可删除。

#### 5.11.3 新增设备规则
- 提供【添加设备】按钮，点击打开“选择设备”弹窗。
- 选择后加入“本次新增”行，可编辑数量与支付金额，可移除。
- 同型号设备允许出现多行（按设备序列号区分），金额可不同。

#### 5.11.4 汇总规则
- 終端數量｜只读汇总｜= 存量设备数量 + 本次新增数量
- 總金額｜只读汇总｜= 存量设备支付金额 + 本次新增支付金额
- 其他字段：終端（自助場景專用）數量、配件數量、每部終端機開戶費、配件申請表、自動打印收據（卡類/非卡類）

#### 5.11.5 设备弹窗
- 搜索：按设备型号、按设备序列号
- 列表字段：设备型号｜设备分类｜设备序列号｜状态｜操作
- 已在“本次新增”中选择的设备显示“已选择”并禁用

### 5.12 相关人员
- 新增人员｜按钮｜追加空白行
- 选择已有人员｜弹窗按钮｜从人员池选择后回填
- 列表字段：人员ID｜姓名｜角色身份类型｜是否主联系人｜电话｜邮箱｜操作
- 操作：删除行

### 5.13 其他信息
- 風控類型｜下拉
- 合約期（开始/结束）｜日期范围

## 6. 校验规则
- 提交审核时：
  - 必填字段（如商戶營業狀態(必須)）不能为空
  - 邮箱字段满足邮箱格式
  - URL 字段需以 `http://` 或 `https://` 开头
  - 数值字段不得为负数
  - 若存在相关人员行，人员ID、姓名、邮箱需合法
- 保存草稿时：
  - 不触发完整必填，但保留基础格式校验

## 7. 底部操作规则
- 保存草稿：提示“草稿已保存（示意）”
- 提交审核：通过校验后提示“提交审核成功（示意）”
- 取消：返回商铺列表页 `shop-list.html`

## 8. 弹窗与联动规则
- 服务弹窗：多选服务并生成可删除Tag
- 设备弹窗：按型号/序列号筛选并回填“本次新增”设备
- 人员弹窗：按ID/姓名/电话/邮箱筛选并回填

## 9. 验收清单
- [ ] 顶部商铺信息卡反显正确
- [ ] 关联企业模块全只读且不可修改
- [ ] 存量设备不可编辑/删除
- [ ] 添加设备弹窗可筛选、选择、回填
- [ ] 同型号可多行且金额可不同
- [ ] 终端数量与总金额自动汇总正确
- [ ] 草稿与提交审核逻辑符合预期
