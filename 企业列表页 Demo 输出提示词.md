## 页面整体结构

页面必须按以下顺序输出，共 **3 个区域**：

1.  筛选区
2.  功能区
3.  列表区

***

## 1. 筛选区

### 1.1 功能定位

企业管理列表页的筛选区，用于帮助运营人员在企业数据量较多时，快速缩小查询范围，定位目标企业，并进入后续查看、编辑、关联查询等操作。

### 1.2 交互形式

筛选区位于企业管理列表页顶部，位于列表内容区上方，筛选区支持【展开 / 收起】交互：

*   默认为收起状态，仅展示两行的核心筛选项
*   点击【展开】后，显示更多补充筛选项
*   点击【收起】后，恢复为默认展示状态
*   同类列表页的【展开 / 收起】按钮位置、顺序、样式需保持一致
*   筛选区采用响应式栅格布局
*   桌面端每行固定展示 3 个筛选项
*   操作按钮区域固定在筛选区右下角
*   日期范围选择器按 1 个完整筛选项宽度处理
*   收起状态共展示 2 行，最多 6 个槽位
*   未用满的槽位保留对齐，不允许自动重排按钮位置
*   收起状态下固定展示以下字段：
    *   名称关键词
    *   地区
    *   企业启用状态
    *   企业审核状态
    *   服务通道
    *   创建时间
*   其余字段仅在展开后显示
*   以上 6 个核心字段默认按顺序排在最前面

### 1.3 操作按钮

筛选区固定提供以下操作按钮，且按顺序排布：

1.  **查询**
2.  **重置**
3.  **展开 / 收起**

其中：

*   【查询】demo 中不需要做真实查询交互
*   【重置】demo 中不需要做真实重置交互
*   【展开 / 收起】必须有交互表现

### 1.4 筛选字段

必须展示以下筛选字段：

| 字段     | 控件类型    | 说明                                     |
| ------ | ------- | -------------------------------------- |
| 名称关键词  | 输入框     | 支持对企业各名称（企业名称，简称，英文名）识别信息进行模糊搜索        |
| 地区     | 下拉单选    |                                        |
| 企业启用状态 | 下拉单选    | 枚举值：启用、停用                              |
| 企业审核状态 | 下拉单选    | 枚举值：待审核、待风控审核、待第三方审核、审核通过、审核不通过        |
| 服务通道   | 下拉多选    | 枚举值：Adyen-AFP、Adyen-payfac、其他（预留）      |
| 创建时间   | 日期范围选择器 |                                        |
| 上單來源   | 下拉多选    | 枚举值：CRM、DMO、SaaS、其他（预留）                |
| 公司模式   | 下拉单选    | 枚举值：直营、加盟、连锁、其他                        |
| 法律地位   | 下拉单选    | 枚举值：个人、个体工商户、有限公司、股份有限公司、合伙企业、非营利组织、其他 |
| 成立时间   | 日期范围选择器 |                                        |
| 公司电话   | 输入框     |                                        |
| 电邮地址   | 输入框     |                                        |

### 1.5 审核流程口径补充

企业审核状态需要兼容以下场景：

*   当所选通道中包含 **非 AFP 通道** 时，企业会进入我方完整审核流程
*   当所选通道 **仅为 AFP** 时，企业可跳过我方风控审核，进入后续 AFP / 第三方处理链路
*   因此企业列表页中需要保留 **待风控审核** 与 **待第三方审核** 两类状态，分别用于表示：
    *   **待风控审核**：等待我方风控继续处理
    *   **待第三方审核**：我方前置审核已完成，当前等待第三方 / 渠道侧审核结果

## 2. 功能区

### 2.1 区域定位

功能区位于筛选区下方、列表区上方，与列表标题同行，按钮靠右排列。

### 2.2 按钮

必须提供以下 3 个按钮：

1.  导出
2.  导入
3.  新增

### 2.3 按钮规则

*   【导出】demo 中不需要做真实导出交互
*   【导入】demo 中不需要做真实导入交互
*   【新增】点击进入新增企业页面
*   按钮顺序必须固定为：**导出 → 导入 → 新增**
*   【新增】按钮固定在最右侧
*   按钮样式需与系统其他主操作按钮保持一致

***

## 3. 列表区

### 3.1 区域定位

列表区分为数据区和操作区两个部分组成，都置于同一行中，数据区从左到右排列数据。\
列表区按表格行展示。\
操作列固定位于最右侧，与数据列属于同一行。\
操作列内多个按钮按纵向排列，顶部对齐显示。\
若某行操作按钮较多导致该行高度增加，则整行所有单元格高度保持一致。

### 3.2 列表显示字段

必须按以下字段展示：

1.  企业名称
2.  企业编号
3.  地区
4.  企业启用状态
5.  企业审核状态
6.  服务通道
7.  公司模式
8.  法律地位
9.  公司电话
10. 电邮地址
11. 成立时间
12. 创建时间
13. 上單來源
14. 创建人
15. 更新时间
16. 操作

### 3.3 布局要求

1.  各列表列按内容设置合理宽度，优先保证主要信息可读性
2.  企业名称列建议宽度 **320-400px**
3.  其他文本列建议宽度 **160-240px**
4.  允许表格出现横向滚动
5.  不允许因列宽过窄导致大部分内容无法识别
6.  文本类单元格内容超长时允许换行展示，最多显示 3 行
7.  超过 3 行的内容以省略号截断，并支持悬浮查看完整内容

### 3.4 状态与操作按钮规则

操作区根据不同状态显示不同按钮：

| 启用状态 | 审核状态   | 列表页操作按钮     |
| ---- | ------ | ----------- |
| 空    | 待审核    | 查看详情        |
| 空    | 待风控审核  | 查看详情        |
| 空    | 待第三方审核 | 查看详情        |
| 启用   | 审核通过   | 查看详情、编辑、停用  |
| 停用   | 审核通过   | 查看详情、编辑、启用  |
| 空    | 审核不通过  | 查看详情、重新提交审核 |

### 3.5 状态补充规则

*   只有审核通过的才会存在启用状态
*   待审核、待风控审核、待第三方审核、审核不通过的数据不会有启用状态
*   当企业审核状态为“待审核”“待风控审核”“待第三方审核”或“审核不通过”时：
    *   企业启用状态为空
    *   不展示启用状态标签
    *   不允许执行启用 / 停用操作
*   仅当企业审核状态为“审核通过”时，列表页才允许展示“启用”或“停用”按钮
*   点击【重新提交审核】后，该企业审核状态变更为“待审核”，启用状态保持为空

### 3.6 操作按钮说明

#### 查看详情

*   点击进入对应状态的企业详情页

#### 编辑

*   点击进入企业编辑页
*   注意：企业编辑页走的是**企业资料修改待审核流程**，不在上单审核流程中

#### 启用 / 停用

*   点击后出现确认弹窗
*   弹窗提示文案：
    *   是否启用该企业
    *   或 是否停用该企业
*   弹窗提供按钮：
    *   关闭
    *   确认
*   点击【关闭】关闭弹窗，不改状态
*   点击【确认】把企业启用状态变成对应状态

#### 重新提交审核

*   再次提交审核
*   将该企业变为“待审核”状态

### 3.7 列表页与详情页职责边界

*   列表页仅提供轻量操作入口
*   审核通过 / 驳回 / 补件等审批类动作统一在详情页执行
*   列表页不承担复杂审批表单和资料区展示职责
