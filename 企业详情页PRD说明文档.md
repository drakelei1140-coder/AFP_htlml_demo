## 2. 页面定位与目标

### 2.1 页面定位
企业详情页是「企业管理」中的核心查看页，用于呈现企业全量档案、风控信息、证件/文件、关联关系（商铺/人员/MID）与关键操作入口（编辑、停用）。

### 2.2 核心目标
1. **一页看全**：支持按模块查看企业主数据与补充信息。
2. **快速检索**：通过顶部锚点导航快速跳转模块。
3. **高频决策支撑**：通过摘要卡、风控模块、时间轴辅助审核/运营判断。
4. **跨实体联动**：可跳转查看关联商铺详情，串联企业-商铺-人员数据链路。

### 2.3 目标用户
- 运营
- 风控
- 审核专员
- 客服/合规
- 管理员

---

## 3. 页面结构总览

页面由以下区域组成：
1. 顶部企业信息卡（企业名、主状态、元信息、操作按钮）
2. 模块导航区（锚点按钮）
3. 快速摘要区
4. 12 个企业信息模块
5. 关联数据区（关联商铺）
6. 修改记录时间轴区
7. 银行账户详情弹窗

---

## 4. 全局交互与通用规则

### 4.1 锚点导航
- 点击导航按钮，平滑滚动到对应模块。
- 页面滚动时，当前可视模块对应按钮高亮（active）。
- 导航与页头均为 sticky，滚动中持续可见。

### 4.2 视觉状态规范
- 使用 Badge 展示状态：如「启用 / 审核通过 / 中风险 / 审核中 / 停用」。
- 文件字段使用 file-chip；图片字段使用 image-chip。

### 4.3 文件/图片操作
- 文件类字段默认展示「文件名 + 上传时间 + 查看按钮」。
- 图片类字段默认展示缩略图 + 查看按钮。

### 4.4 列表分页说明
- 相关人员：页脚展示「第 X / Y 页（每页最多10个相关人员）」。
- 关联商铺：页脚展示「第 X / Y 页（每页最多10个商铺）」。

### 4.5 弹窗规则（银行账号）
- 从「关联银行账号」卡片点击后打开。
- 弹窗内展示该账号完整字段（键值对网格）。
- 支持关闭按钮与点击遮罩关闭。

## 6. 状态与文案规则（建议）

1. **状态字段统一 Badge 组件**：成功/警告/危险颜色语义统一。
2. **空值统一展示**：建议统一为 `-`，避免空白。
3. **非必填字段标识**：保留 `(非必須)` 后缀，避免误解。
4. **链接字段**：建议支持点击跳转并做 URL 安全校验。
5. **重复字段治理**：企业联系中「接收營銷資訊」出现两次，建议在后续版本明确区分（如短信营销/邮件营销）。

## 7. 数据口径与接口建议（供研发参考）

1. 企业详情接口建议按模块分组返回，便于前端按 section 渲染。
2. 文件类字段建议统一结构：
   - fileName
   - fileUrl
   - uploadedAt
   - fileType
3. 图片字段建议统一结构：
   - thumbUrl
   - originUrl
   - uploadedAt
4. 关联列表（商铺/人员）建议独立分页接口，避免企业详情接口过大。
5. 时间轴建议返回 eventType + eventDesc + operator + createdAt，支持后续筛选。
