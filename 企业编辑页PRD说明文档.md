# 企业编辑页 PRD 说明文档（详细版）

## 1. 文档信息
- 适用页面：`enterprise-edit.html`
- 页面定位：企业存量资料编辑页（保存修改/提交审核）
- 更新时间：2026-04-26

## 2. 页面目标
1. 支持已存在企业信息修改。
2. 明确“可编辑字段”和“只读字段”边界。
3. 通过脏数据识别（isDirty）控制提交审核动作，避免无变更提交。

## 3. 页面结构
1. 顶部信息区（返回、审核状态、企业名+CID）
2. 模块导航（12个模块）
3. 表单区（1-12模块）
4. 底部操作区（保存修改 / 提交审核 / 取消）
5. 群组弹窗
6. 银行账号详情弹窗

## 4. 全局交互与状态规则
- 锚点导航平滑滚动 + 当前模块高亮。
- `submitBtn` 初始禁用；表单任一字段变更后置为可用。
- 保存修改后重置脏状态（`isDirty = false`）。
- 点击提交审核前：
  1) 必须 `isDirty=true`
  2) 二次确认弹窗（“确认提交审核？”）
  3) 通过校验后提交
- 顶部展示当前审核状态（示例：审核通过）。

## 5. 模块与字段清单

> 字段格式：**字段名｜控件类型｜编辑权限｜说明**

### 5.1 主要数据
- CID｜只读框｜只读｜系统生成
- 上級公司ID｜输入框｜可编辑
- 企业营业状态｜下拉｜可编辑（必填）
- 企业类型｜输入框｜可编辑（必填）
- 关联银行账号｜按钮+只读提示｜只读｜来自关联商铺/MID，点击查看详情弹窗
- 开通Kpay服务｜下拉｜可编辑（必填）
- 关联群组｜弹窗多选 + Tag｜可编辑（必填）

### 5.2 企业名称
- 与新增页一致，共11个字段，支持编辑。

### 5.3 企业地址
- 与新增页一致，共9个主地址字段 + 结构化地址拆分输入组。
- 支持编辑。

### 5.4 企业主体/经营
- 与新增页一致，共13字段。
- 支持编辑。

### 5.5 企业基础
- 字段集合与新增页一致。
- 差异点：
  - 第三方通道品牌名（渠道）/服务：只读展示（AFP返回字段，不可编辑）。

### 5.6 风控/业务
- 字段集合与新增页一致。
- 差异点：文件字段支持显示“已上传文件”Tag，可点击“删除旧文件”。

### 5.7 企业联系
- 与新增页一致。
- 電郵地址为必填。

### 5.8 企业证件
- 与新增页字段一致。
- 差异点：已有文件以“已上传文件”Tag展示，可删除旧文件。

### 5.9 企业文件
- 与新增页16类文件字段一致。
- 差异点：每个文件项均支持“已上传文件”Tag + 删除旧文件。

### 5.10 线上资料
- 与新增页一致。

### 5.11 相关人员
- 表格字段：人员名称、角色类型、是否主联系人、电话、邮箱、操作。
- 支持新增/删除。
- 删除后会触发脏状态。
- 至少保留1条。

### 5.12 其他
- 企业logo：图片上传 + 预览。
- 私隱政策：文件上传 + 已上传文件Tag + 删除旧文件。
- 创建时间、更新时间：只读展示（系统生成）。

## 6. 银行账号详情弹窗（编辑页特有）
- 入口：主要数据模块中的“SID/MID”按钮。
- 展示字段：
  - 結算週期
  - 結算幣種
  - 歸屬國編號
  - 歸屬省編號
  - 是否對公
  - 收款銀行賬戶名稱
  - 銀行名稱
  - 銀行號碼
  - 銀行編號
  - 分行號
  - 卡類型
  - 結算身份證號
- 所有字段在弹窗中只读展示。
- 支持“关闭”按钮与点击遮罩关闭。

## 7. 校验规则
- 必填校验：`required` 字段不能为空。
- 邮箱校验：`email-field`、`person-email` 必须为合法邮箱。
- 数字校验：`number-field` 必须是可解析数值。
- 相关人员：至少1条。

## 8. 底部操作规则
- 保存修改：可直接保存（不强制校验），提示“修改已保存（示意）”。
- 提交审核：需有变更 + 二次确认 + 表单校验通过。
- 取消：返回企业列表。

## 9. 关键差异（相较新增页）
1. 顶部元信息改为“审核状态 + 企业名/CID”。
2. 增加“关联银行账号”只读查看能力。
3. 增加“已上传文件”存量展示及删除旧文件。
4. 第三方通道品牌名字段改为只读。
5. 提交审核按钮受 isDirty 状态控制。
